windy-pro-mobile — ENDPOINT INVENTORY (Wave-7 adversarial audit)

Mobile is a client. It has two kinds of "endpoints":
   A. In-app routes (expo-router files under src/app/).
   B. External HTTP calls it makes to other ecosystem services.

Auth gating (in-app — not strict; each screen decides):
  /(tabs)/mail      loads native list only when identityApi.isAuthenticated()
  /(tabs)/ecosystem shows "Sign in" CTA if not authed
  /cloud/files      redirects to /auth/login when not authed
  /hatch            requires identityApi session (calls agent-provision endpoint)
  /settings/trust   reads ecosystem passport; empty state if not set
  all other screens are unauthenticated-reachable (either show placeholder or
  call a feature that degrades).
